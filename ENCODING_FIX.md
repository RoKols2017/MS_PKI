# Исправление кодировки PowerShell скриптов

## Проблема

Все PowerShell скрипты (.ps1 и .psm1) в проекте сохранены без BOM (Byte Order Mark), что может вызывать проблемы с отображением русских символов на Windows.

## Решение

### Шаг 1: Закройте все PowerShell файлы в редакторе

Перед исправлением кодировки закройте все открытые .ps1 и .psm1 файлы в Cursor/редакторе.

### Шаг 2: Запустите скрипт исправления

```powershell
powershell -ExecutionPolicy Bypass -File "fix_encoding_final.ps1"
```

### Шаг 3: Проверьте результат

```powershell
powershell -ExecutionPolicy Bypass -File "check_encoding.ps1"
```

Все файлы должны показать `BOM=True`.

## Что было исправлено

1. Все .ps1 и .psm1 файлы теперь имеют UTF-8 с BOM
2. Все места в коде, где используется `Out-File -Encoding UTF8`, уже корректны (создают UTF-8 с BOM)

## Файлы, требующие исправления

- `src\lib\*.psm1` (5 файлов)
- `src\*.ps1` (2 файла)
- `src\pki-*\*.ps1` (4 файла)

## Примечание

После исправления кодировки все русские символы в скриптах будут корректно отображаться в PowerShell и редакторах.
